## Grammarly 接受哪些支付方式？先看官方怎么说

在折腾虚拟卡之前，得先搞清楚 Grammarly 的"门槛"在哪里。根据 Grammarly 官方支持文档，目前接受的支付方式包括：

- **信用卡 / 借记卡**：Visa、Mastercard、American Express、Discover、UnionPay（银联）。关键是——卡片必须支持循环电商交易（recurring e-commerce transactions），也就是能扣月费、能自动续费。
- **预付卡（Prepaid Card）**：Visa、Mastercard、American Express、Discover、UnionPay 预付卡**理论上**也可以，但前提是发卡行批准 Grammarly 的授权请求。换句话说，能不能成，要看发卡行脸色。
- **PayPal、UPI、Apple Pay、Google Pay**：网页端可用 PayPal，移动端 App 升级仅支持 Apple Pay / Google Pay。

不支持的方式：Visa Electron、Mastercard Maestro、Interac、个人支票、汇票、电话支付、现金。

**重点翻译一下这段话**：UnionPay 银联卡 Grammarly 是接受的，但国内大部分银联卡默认不开通境外循环扣款权限；预付卡"理论支持、实际看发卡行"，这就给虚拟卡留下了空间——只要卡段（BIN）干净、发卡行配合授权，就能过。

## 为什么国内信用卡订阅 Grammarly 总是失败？

先把失败原因列清楚，对症下药：

1. **未开通境外电商交易**：很多国内卡默认关闭"境外无卡支付"，需要打电话给银行单独开通，开通后还可能被风控。
2. **不支持循环扣款**：Grammarly 是订阅制，每月 / 每季 / 每年自动扣款，部分国内借记卡不支持 recurring，第一次能过，第二次续费就挂。
3. **3D Secure 验证问题**：国内卡走 3DS 验证时弹短信、跳银行页面，Grammarly 的支付通道有时候接不住这个跳转，直接判失败。
4. **地址不匹配**：账单地址填中文拼音、和卡的真实持卡人地址对不上，风控直接拒。
5. **卡段被风控**：某些 BIN 段被 Stripe / Adyen 这类支付网关拉黑，无论怎么填都过不了。
6. **地区限制**：从中国大陆 IP 直接订阅，部分账号会被标记，付款环节直接走不下去。

虚拟卡之所以能解决问题，是因为它本质上是海外银行 / 牌照机构发行的 Visa / Mastercard，BIN 段是海外段，账单地址是海外地址，天然绕开了上面 1、2、3、4、5 这五条。

如果填完卡还是被拒，按这个顺序排查：

2. **账单地址对不对**：PokePay App 里的卡详情页有专门的账单地址，一字不差填到 Grammarly。
3. **IP 干不干净**：换美国 / 香港 / 新加坡住宅 IP，别用免费代理，免费 IP 经常被风控。
4. **卡段是不是被拉黑**：极少数情况下 PokePay 的某个 BIN 段会被 Stripe 暂时风控，换个时间段重试，或者联系 PokePay 客服换一张新卡段。

6. **账号地区**：Grammarly 账号注册时如果选了受限地区，付款可能直接失败，新注册一个账号用海外 IP 注册更稳。
